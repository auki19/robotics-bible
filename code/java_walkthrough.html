1. Java Introduction

This first part of the guide is for complete beginners to Java, so if you already get the gist of it skip to part 2 for applying it to FTC. Follow along with this portion of the tutorial with an online Java compiler like CodeHS. If you have a personal computer rather than a school laptop I'd advise downloading an IDE like IntelliJ since it's much faster than online ones. We won't be getting to Java coding for FTC specifically just yet, but we'll switch over to the FTC OnBotJava application in the second section of this guide. Keep in mind that this guide will only teach you Java skills that are relevant to FTC, and won't be getting to certain otherwise important aspects of the language.

1.1 Printing

Opening up the compiler you'll see a screen that looks something like this:

public class MyProgram { public static void main(String[] args) { System.out.println("Hello, World!"); } }

For now, just ignore everything other than the System.out.println("Hello, World!"); line. We'll go over what public class MyProgram and public static void main(String[] args) mean later in this section, but for now just know that all your code should be contained within those brackets.
Click the blue Run button in the upper right hand corner of your program window and notice that the phrase Hello, World! contained within the quotation marks are printed into the output window on the right side of your screen.
Next, try copy-pasting the line or changing the text like this:

public class MyProgram { public static void main(String[] args) { System.out.println("Hello, World!"); System.out.println("Goodbye, World!"); } }

The output window should show:

Hello, World! Goodbye, World!

When you hit run, the Java compiler reads the lines of code you write from top to bottom and executes them in that order. System.out.println() is a line of code that takes a phrase within quotations, called a String, inside its parenthesis and prints it into the output. The specifics of how this line works will make more sense once you make it to reviewing methods, but until then that's all you need to know.
Strings aren't the only things System.out.println() statements can print. Try removing the quotation marks and just type a number, for example System.out.println(23);. When the code executes, 23 should be printed in the console.
You may have noticed that every line of code within main ends with a semicolon. This tells the Java compiler where the statement ends, and it's very important to make sure not to forget to put it there. When you're working with a huge script and hundreds of lines it can be very irritating to search through it all and find an error, especially when it's something as little as a singular semicolon.

1.2 Variables

Integers

Variables in Java function very similarly to what you know from algebra class. To create a variable, you must first enter the type, then the name, then set it equal to a value. For example, an integer value is represented by the variable type int. To create an int variable named x that equals 0, enter this:

public class MyProgram { public static void main(String[] args) { int x = 0; } }

Next, try printing x with a System.out.println() statement like this:

public class MyProgram { public static void main(String[] args) { int x = 0; System.out.println(x); } }
Output (Click to reveal)
0

Because the = operator means assigning the value on the right of the = sign to the variable on the left of it, the code should output 0, the assigned value of x.
Math can be done with variables, too. Note that while initially creating a variable you must define its type, but when referencing or modifiying it at any point after that you use the variable name only.

public class MyProgram { public static void main(String[] args) { int x = 1; int y = 2; int sum = x + y; System.out.println(sum); } }
Output
3

Subtraction (-), multiplication (*), and division (/) can all be done in this same way.
If you want to increment or decrement a variable you can write something like this:

public class MyProgram { public static void main(String[] args) { int number = 0; number = number - 5; System.out.println(number); } }
Output
-5

Because you're setting number equal to itself minus 5 and its initial value is 0, your code should output -5.
However, rather than rewriting the variable name every time you want to change it, you can just type the line as number -= 5;. For another example, typing number *= 2; is the same thing as typing number = number * 2;

CHALLENGE: Try determining the output of this code segment without the use of a compiler.

public class MyProgram { public static void main(String[] args) { int a = 1; a = a + 8; a /= 2; a += -3; a = a * 5; System.out.println(a); } }
Output
5

Doubles

You may have noticed an issue with int values--that is, dividing with them doesn't leave any decimal or remainder value. For example, even though dividing 3 by 2 should equal 1.5, dividing an int equal to 3 by an int equal to 2 will cut off the decimal place and return the value 1.
ints aren't the only kinds of variables, however. doubles are variables capable of holding decimal places as well. Regardless of whether a double has a value in its decimal places, there must be at least one decimal place when declaring one. To solve the issue of division with ints, use these variables instead:

public class MyProgram { public static void main(String[] args) { double quotient = 3.0 / 2.0; System.out.println(quotient); } }
Output
1.5

Strings

Another useful variable type is the String. Like you saw earlier in the first lesson, a String is a kind of variable that is contained within quotation marks that can hold text. Take this code segment for instance:

public class MyProgram { public static void main(String[] args) { String str = "First Tech Challenge"; System.out.println(str); } }
Output
First Tech Challenge

More text can be added to Strings using += as well.

public class MyProgram { public static void main(String[] args) { String str = "First Tech Challenge"; str += " 2023"; System.out.println(str); } }
Output
First Tech Challenge 2023

Booleans

Another important kind of variable is a boolean. boolean values store either true or false.

public class MyProgram { public static void main(String[] args) { boolean value = true; System.out.println(value); } }
Output
true

booleans come with a few different operators. && (pronounced "and") is an operator that compares two different boolean values and returns true if and only if both booleans are true.

public class MyProgram { public static void main(String[] args) { boolean x = true; boolean y = true; boolean value = x && y; System.out.println(value); boolean z = false; System.out.println(value && z); } }
Output
true false

|| (pronounced "or") operators return true if either one of the two values being compared are true. Otherwise, it returns false.

public class MyProgram { public static void main(String[] args) { boolean x = false; boolean y = false; boolean value = x || y; System.out.println(value); boolean z = true; System.out.println(value || z); } }
Output
false true

Some operators also compare non-boolean values and return a boolean. > (greater than) and < (less than) operators compare two int or double values and return true if the comparison is valid. >= (greater than or equal to) and <= (less than or equal to) are also useful operators that compare two values. == (equals-equals) returns true the two values being compared are identical, and != (not-equals) returns true if they're different.

public class MyProgram { public static void main(String[] args) { int value1 = 7; int value2 = 4; int value3 = 4; System.out.println(value1 > value2); System.out.println(value1 < value3); System.out.println(value2 >= value3); System.out.println(value1 <= value2); System.out.println(value2 == value3); System.out.println(value2 != value3); } }
Output
true false true false true false

Finally, the ! (pronounced "not") operator negates a boolean value.

public class MyProgram { public static void main(String[] args) { boolean p = true; System.out.println(!p); } }
Output
false

Variables are vital to coding in Java (and coding languages in general), so it's important to understand this lesson before moving on to the next.

1.3 If Statements

if statements are control flow tools that can change what happens when the code is executed based on a boolean value. If the value within its parenthesis are true, the code within the if statement's brackets will run. If not, then it will not run.

public class MyProgram { public static void main(String[] args) { boolean willRun = true; if (willRun) { System.out.println("test"); } if (!willRun) { System.out.println("test2"); } } }
Output
test

an else statement can be placed directly after the brackets of an if statement. The contents of the else statement only run if the if statement doesn't.

public class MyProgram { public static void main(String[] args) { boolean a = true; boolean b = false; if (a && b) { System.out.println("if"); } else { System.out.println("else"); } } }
Output
else

Finally, else if statements only run if the previous if statement doesn't AND the boolean value within the parenthesis is true.

public class MyProgram { public static void main(String[] args) { int number = 3; if (number == 1) { System.out.println("one"); } else if (number == 2) { System.out.println("two"); } else if (number == 3) { System.out.println("three"); } else if (number == 4) { System.out.println("four"); } } }
Output
three

1.4 Loops

Loops can be used to run a certain line of code a certain amount of times without having to rewrite it over and over. A while statement runs for however long the boolean value within its parenthesis remains true.

public class MyProgram { public static void main(String[] args) { int i = 0; while (i < 3) { System.out.println("Looped " + i + " times"); i++; } System.out.println("Because i is no longer less than 3, the while statement stopped looping."); } }
Output
Looped 1 times Looped 2 times Looped 3 times Because i is no longer less than 3, the while statement stopped looping.

while loops are generally used when the number of times it needs to loop is indefinite (while waiting for user input, looping infinitely, etc.). Loops that loop a set number of times like the example above should be rewritten as for loops.
A for loop has three conditions within its parenthesis as opposed to one in if and while statements. The first condition is creating a 'counter' variable which usually begins at zero. The second condition is setting an end point for the looping with a boolean expression using the counter variable, just like how you do it in a while statement. The third condition is for incrementing the counter at the end of each loop. for loops may look complicated, but they're pretty simple once you're used to them and are very important for condensing while loop code.

Here's the previous example rewritten as a for loop:

public class MyProgram { public static void main(String[] args) { for (i = 0; i < 3; i++) { System.out.println("Looped " + i + " times"); } System.out.println("Because i is no longer less than 3, the while statement stopped looping."); } }
Output
Looped 1 times Looped 2 times Looped 3 times Because i is no longer less than 3, the while statement stopped looping.
